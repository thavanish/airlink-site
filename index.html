What is AirLink?

AirLink started as a solution to a common problem: existing game server panels were either too complex, too limited, or both. We wanted something that worked well out of the box but could grow with your needs.

The panel is built on modern tools like TypeScript and Prisma. It handles the basics like user management, server monitoring, and resource allocation. What makes it different is the addon system, which lets you add features without modifying core code.

Right now it's in active development with a small community using it for various game servers. The architecture is stable enough for production use, though we're still working on polish and additional features.

It's MIT licensed, so you can use it however you need to. Fork it, modify it, deploy it commercially. The community has already started building addons for things like Minecraft mod management and billing integration.

Core Features

Server Management

Deploy and manage multiple game servers from a single interface. Start, stop, restart, and monitor resources in real-time.

User System

Handle permissions properly. Admin controls, user roles, and server-level access management work how you'd expect.

Addon Architecture

Extend functionality through addons. They can add routes, database tables, UI components, and integrate external services.

Database Migrations

Addons handle their own database setup. Migrations run automatically when an addon is enabled.

Modern Stack

TypeScript for type safety, Prisma for database queries, Express for routing. Standard tools, straightforward codebase.

REST API

Everything the panel does is accessible through the API. Build your own interfaces or automate deployments.

Installation

Two ways to get started. The installer handles most of the setup automatically.

Quick Install

Run the installer as root. It sets up the panel, configures systemd, and handles dependencies.

bash <(curl -s https://raw.githubusercontent.com/airlinklabs/panel/refs/heads/main/installer.sh)

After installation, control the panel with systemd:

systemctl start airlink-panel systemctl stop airlink-panel

Manual Install

1

Clone Repository

cd /var/www/ git clone https://github.com/AirlinkLabs/panel.git cd panel
2

Set Permissions

sudo chown -R www-data:www-data /var/www/panel sudo chmod -R 755 /var/www/panel
3

Install Dependencies

npm install -g typescript npm install --omit=dev
4

Database Setup

npm run migrate:dev
5

Build and Run

npm run build-ts npm run start

Requirements

  • Node.js v16 or higher
  • npm v8 or higher
  • PostgreSQL or MySQL
  • Git

Production Deployment

For production, use pm2 to keep the panel running:

npm install pm2 -g pm2 start dist/app.js --name "panel" pm2 save pm2 startup

Addon System

Addons let you extend the panel without touching core code. They can add new pages, database tables, API endpoints, and UI components.

How Addons Work

Each addon lives in its own directory under panel/storage/addons/. When enabled, the addon gets access to the router, database, and various utilities.

Addons can register sidebar items, add server menu entries, create their own database tables, and hook into the panel's lifecycle. The system handles loading, initialization, and cleanup automatically.

Basic Structure

my-addon/ ├── package.json ├── index.ts ├── views/ │ └── main.ejs └── lib/ └── helpers.ts

What Addons Can Do

Add Routes

Define new endpoints and pages. Handle GET/POST requests and serve custom views.

Database Access

Run migrations to create tables. Use Prisma for queries or run raw SQL when needed.

Modify UI

Add items to the sidebar, server menu, or create new sections. Use existing components or build new ones.

External Integration

Connect to external APIs, payment processors, mod repositories, or any other service.

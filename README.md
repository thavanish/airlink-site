# AirLink Docs Site

Documentation and landing page for [AirLink Panel](https://github.com/AirlinkLabs/panel).

## Quick Edit Guide

All site-wide settings live in **`config.json`** at the root of this repository.
Edit that one file to change:

| What                         | Where in config.json                  |
|------------------------------|---------------------------------------|
| Site title / version         | `site.title`, `site.versionLabel`     |
| GitHub repos                 | `site.githubPanel`, `site.githubDaemon` |
| Discord link                 | `site.discordUrl`                     |
| Under-construction mode      | `underConstruction.enabled` (true/false) |
| Under-construction messages  | `underConstruction.*`                 |
| Install commands             | `install.quickInstallCommand`, `install.manualSteps` |
| Feature cards                | `features[]`                          |
| Community addons             | `addons[]`                            |
| Docs sidebar manifest        | `docs.manifest[]`                     |
| Team / contributor overrides | `team.customContributors`             |

## Adding a New Community Addon

1. Add an entry to the `addons` array in `config.json`
2. Create a folder `public/assets/addons/<your-addon-id>/`
3. Drop images (`.jpg`, `.png`, etc.) and optionally an `icon.png` into that folder
4. Push to `main` — the GitHub Actions workflow will auto-generate `manifest.json`

## Adding Feature Screenshots

Drop images into `public/assets/features/<feature-id>/` and push.
The workflow auto-generates `manifest.json` so the slideshow picks them up.

## Under-Construction Mode

Set `underConstruction.enabled` to `true` in `config.json` to:
- Show a popup on page load
- Add a badge to the navbar
- Show a note in the footer
- Display a message in the loading screen

## GitHub Actions Workflow

Located at `github/workflows/generate-manifests.yml`

> **Note for deployment:** rename the `github/` folder to `.github/` on your server or in your
> fork. It is named `github/` in this archive because `.github/` directories are excluded when
> creating `.zip` archives on Linux with standard tools.

The workflow:
1. Scans `public/assets/` and generates `manifest.json` in each image folder
2. Pre-fetches GitHub API data (contributors, commits, stars) using the built-in `GITHUB_TOKEN`
   and saves the results to `public/api-cache/` so visitors are never rate-limited

It runs on every push to `main` and every 6 hours.

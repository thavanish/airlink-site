# Contributing to AirLink Panel Website

Thanks for considering contributing to the AirLink Panel website. Here's how you can help.

## Ways to Contribute

### Documentation
- Fix typos or unclear explanations
- Add examples to existing guides
- Improve code snippets
- Translate content

### Design
- Improve visual design
- Enhance mobile responsiveness
- Fix UI bugs
- Suggest UX improvements

### Content
- Write new guides
- Update outdated information
- Add missing sections
- Improve navigation

## Getting Started

1. Fork the repository
2. Create a branch for your changes
3. Make your modifications
4. Test locally
5. Submit a pull request

## Pull Request Process

1. Make sure your changes work on different browsers
2. Test on mobile devices if you changed layout
3. Keep pull requests focused on a single change
4. Write a clear description of what changed and why
5. Reference any related issues

## Code Style

- Use semantic HTML
- Keep CSS organized by component
- Write readable JavaScript
- Follow existing naming conventions
- Comment complex logic

## Testing

Before submitting:
- Test on Chrome, Firefox, and Safari
- Check mobile responsiveness
- Verify all links work
- Ensure images load properly

## Questions?

If you're unsure about something, open an issue to discuss it before spending time on implementation.
